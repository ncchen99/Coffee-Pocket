# 店家詳細頁 · Cafe Detail

## 目的

讓使用者在「看完一張詳細資訊頁」後,能在 10 秒內判斷:
**這間現在能不能去、合不合我此刻的情境**。
不追求展示華麗的店家介紹,而是回答「決策題」。

## 入口

- Mobile 結果列表 / 地圖 pin 點擊
- Desktop 結果列表項目點擊 → 右側替換成詳細頁,或 Modal
- 口袋名單點擊
- 「今天去哪」單卡的「就去這間」

## Desktop 差異

- 從結果列表點進後,**地圖維持在右側**;詳細頁在中間欄、840 寬左右
- 主圖改為 2x2 縮圖網格 (而非 carousel)
- 「位置」區塊省略,因為右側地圖已有
- footer CTA 改為頁面最上方的 inline 按鈕

## 互動 / 狀態

- **♡ 收藏按鈕**:未登入 → 引導去 [login](login.md);已登入 → 直接 toggle
- **AI 摘要**:長按 / hover 顯示「這個摘要從哪些評論萃取」(來源列表)
- **標籤**:每顆標籤可點 → 顯示 evidence (「3 則評論提到插座」),也可投票 👍 / 👎 修正
- **營業時間**:預設只顯示今天,可展開全週
- **「現在營業」綠燈**:伺服器時間 + 店家時區計算,而非 client 端
- **修正資訊**:進入 [cafe-edit](cafe-edit.md)

## 邊界狀態

- 永久歇業 → 整頁加上灰色遮罩 + 紅字「已歇業」,CTA 改為「回報還開著」
- 暫停營業 / 公休 → 黃色提示條
- 沒有照片 → 主圖區顯示手繪 placeholder (沿用 wireframe 風格),不顯示空灰塊
- 沒有 AI 摘要 (資料不足) → 整段隱藏,不顯示「暫無摘要」(降低噪音)
