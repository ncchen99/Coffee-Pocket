# 登入頁 · Login

## 目的

降低登入摩擦,不強迫使用者開帳號才能用基本功能。
**只有「收藏 / 編輯店家 / 投票標籤」**才需要登入,因此這頁通常是被動觸發的 modal,而非主動入口。

## 入口

- 點 ♡ 收藏 (未登入時)
- 進入 [cafe-edit](cafe-edit.md) / [cafe-add](cafe-add.md)
- 標籤投票 👍 / 👎
- 個人頁主動「登入」按鈕

## 互動

- Google OAuth 走 Supabase Auth 的 OAuth flow,在系統瀏覽器 / WebView 完成
- Apple Sign-In iOS 必備 (App Store 規則),Web 端可隱藏
- 登入完成後**回到觸發登入的頁面**並繼續原本動作 (e.g. 自動完成收藏)
- ✕ / 「返回不登入」:回到原頁,觸發的動作丟棄

## 邊界狀態

- OAuth 失敗 → inline 顯示「登入失敗,再試一次」,不換頁
- 已被封禁的帳號 → 顯示客服信箱
- 第一次登入 → 直接進入帳號;不做額外的 profile 設定步驟 (盡量不打斷)
