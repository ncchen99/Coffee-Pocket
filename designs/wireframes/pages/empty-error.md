# 空狀態 / 錯誤頁 · Empty & Error States

## 目的

把「沒結果 / 失敗 / 還沒資料」做成一致的視覺語言。
**永遠提供一個下一步**,不顯示死路。
沿用手繪 ann 風格,讓錯誤不至於有挫敗感。

## 通用結構

```
┌──────────────────────────────┐
│                              │
│        ✎ (手繪小插圖)         │  60–80px,Pencil emoji 或 SVG
│                              │
│        標題 (一句話)          │  h-title
│        說明 (一句話 meta)     │
│                              │
│      〔 主要動作 〕            │  primary
│      〔 次要動作 〕            │  ghost / link
│                              │
└──────────────────────────────┘
```

## 各場景文案 / 動作

### 1 · 搜尋無結果

- 圖:空咖啡杯
- 標題:**沒有完全符合的店**
- 說明:你要的條件太多了,試試放寬其中一項
- 主要:〔 移除「安靜」(會多 8 間) 〕  ← 系統自動找最容易放寬的條件
- 次要:〔 看接近的 5 間 〕  ← 顯示部分符合的結果

### 2 · 定位失敗 / 拒絕

- 圖:斷掉的指南針
- 標題:**找不到你在哪**
- 說明:沒關係,告訴我你想以哪裡為中心
- 主要:〔 用「臺南 · 中西區」當中心 〕
- 次要:〔 再試一次定位 〕

### 3 · 網路錯誤

- 圖:訊號 / 雲圖示
- 標題:**連不上**
- 說明:檢查網路後再試一次
- 主要:〔 重新載入 〕
- 次要:〔 查看離線收藏 〕  ← 還是能進 [pocket-list](pocket-list.md) 看快取

### 4 · 伺服器錯誤 (5xx)

- 圖:打翻的咖啡
- 標題:**我們這邊出了點事**
- 說明:工程師已收到通知,30 秒後再試
- 主要:〔 重新整理 〕
- 次要:〔 [回報問題](mailto:) 〕

### 5 · 沒有口袋 / 沒有貢獻

- 見 [pocket-list](pocket-list.md) 與 [profile](profile.md) 各自描述

### 6 · 永久歇業 / 已合併到另一筆

- 圖:鎖
- 標題:**這間店已歇業**
- 說明 (歇業):資料於 2026-03 由社群回報
- 主要 (歇業):〔 回報還開著 〕
- 說明 (合併):這間店已合併到「窩 café (新店)」
- 主要 (合併):〔 看新的那間 〕

## 共用設計原則

- **不要技術細節**:不顯示 stack trace、不顯示 HTTP code (除非 dev mode)
- **不要罪惡感文案**:避免「你的輸入錯誤」,改說「條件太多了」
- **保留導航**:即使全頁錯誤,**頂部 / 底部分頁仍可用**,讓使用者切到別頁
- **手繪 ann 的圖示**統一從同一組 icon set 來,確保風格一致
